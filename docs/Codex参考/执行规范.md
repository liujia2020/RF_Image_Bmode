# 验证规则与项目纪律

> 给线二的 Codex。最后更新: 2026-06-12。

## 运行组织

每个正式运行按四个阶段组织：

```text
experiments/dual_input_v1/<YYYY-MM-DD_short_name>/
├── 01_config/          训练前配置 (README / config.yaml / manifest)
├── 02_train/           训练产物 (checkpoints / probes / logs / metrics)
├── 03_validate/        验证结果 (figures / metrics / verdict)
└── 04_proma_verify/    Proma 独立验证
```

## Smoke Test

只回答：
- 代码能跑吗？
- 有 NaN 吗？
- 有 OOM 吗？
- loss 能正常计算吗？

**不能证明质量。**

## 大规模数据处理铁律

超过 100 文件或预计 >5 分钟的任务：
1. **点火测试**：3~5 文件验证语法/NaN/shape
2. **全量执行**：通过后全量，输出进度到日志
3. **完整性校验**：确认数量和内容

## 训练探针输出

每次训练运行必须产出：
- **stability.csv**：每 epoch 记录 loss、SSIM3D、L1
- **每 epoch 训练探针图**：`02_train/probes/train/train_epochXXX_y16.png`
- **每 epoch 验证探针图**：`02_train/probes/val/val_epochXXX_y16.png`
- **loss 曲线**
- **summary.md**

### 每 epoch 探针图格式

每张探针图固定为 `3 行 × 3 列`：

```text
         baseline    pred    gt
carotid    [图]      [图]    [图]
muscle     [图]      [图]    [图]
phantom    [图]      [图]    [图]
```

要求：
1. 默认只输出一个切片：`y=16`。
2. train 与 val 每个 epoch 各输出一张合并图。
3. 三个类别样本在训练开始前固定，并写入 `01_config/manifest` 或 `summary.md`，保证跨 epoch 可比。
4. 列顺序固定为 `baseline-bmode | pred | gt-bmode`。
5. 该探针只用于观察训练状态和发现异常；Codex 只报数字和现象，不宣称图像质量通过。
6. 如 checkpoint 不是每 epoch 保存，仍必须在每个 epoch 结束时用当前模型即时生成探针图。
