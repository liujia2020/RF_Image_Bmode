## Phase 3：双支路 B-mode 网络实现与验证（完成 ✅）

**时期**：2026-06-13

**背景**：通道顺序锁定 + 显存定为本地 8G 后，Claude 出双支路架构规格，Codex 实现，Proma 独立验证（执行/验证分离，铁律第 1 条）。

**架构**：`DualBranchBModeUNet`（`rf_cgan_models.py`，7.34M 参数）
- RF 支路：1×1×1 Conv 把 1536→64 立即降维（省显存关键，不依赖通道顺序）
- B-mode 支路：3×3×3 Conv 1→32
- 早期融合（concat）→ 3 次各向同性下采样编码器 → 解码器（trilinear 上采样 + skip）
- 输出头 1×1×1 Conv，bias=-2.0（防灰雾），sigmoid → [0,1]
- 全 BatchNorm3d(momentum=0.9)，关 AMP
- 损失（`rf_cgan_losses.py`）：0.84·(1−SSIM3D) + 0.16·L1，SSIM win=7, data_range=1

**Codex 自测 + Proma 独立验证（9 项交叉一致）**：

| 验收项 | 实测 |
|--------|------|
| 输出 shape | [2,1,64,32,32] ✓ |
| 输出范围 | [0,1] ✓ |
| 输出头 bias | −2.0 ✓ |
| BN / IN / momentum | 22 / 0 / 0.9 ✓ |
| 参数量 | 7,335,649（7.34M，轻量） |
| backward + 梯度 | 成功，无 NaN/Inf ✓ |
| 损失自洽 | pred=gt→≈0（5e-8）；pred_rand→0.89 ✓ |
| 显存（batch=2 fp32 fwd+bwd） | 3.42G allocated / 4.36G reserved（<< 8G）✓ |
| 数值 | NaN=0 Inf=0 ✓ |

**值得注意**：
- 显存 Proma 实测 3.42G vs Codex smoke 1.74G（测量点不同）；训练以 3.42G 为基准更保守，加 Adam 状态 ~0.12G，预期单步 <4G，8G 充足。
- Codex 随机输入下 output mean=0.618（随机权重对随机输入的响应，非真实数据）；bias=−2 的防灰雾效果需训练首个 epoch 看真实 pred_mean 验证。

**结论**：模型代码**工程正确性 PASS**（实现符合规格、显存达标、可进入训练）。**注意：这不是图像质量 PASS**——质量待训练出图 + 人眼 Slicer 签收（验证门禁）。

**文件**：`rf_cgan_models.py`、`rf_cgan_losses.py`、`scripts/smoke_test_model.py`（Codex）、`scripts/verify_model_proma.py`（Proma 独立）。

## Phase 2b：1536 通道内部顺序查证（完成 ✅）

**时期**：2026-06-13

**背景**：线一交接文档里写了 `3角度 ×(RC/CR实虚)× 128通道` 的假设，但从未有人回溯原始 matlab 验证。写架构前必须锁死。

**怎么做**：Claude 直接从 GitHub 抓取原始 matlab 生成脚本 `build_RF_random64_full1500.m`，对照线一 reader 代码和物理自洽性，三重交叉印证。

**结论**：

```
1536 = 128 接收阵元 × 3 角度 × 4 分量（RC_real, RC_imag, CR_real, CR_imag）
C-order：通道最外（最慢变）→ 4 分量最内（最快变）

c = ch*12 + ang*4 + comp
[1536] 无损 reshape → [128, 3, 4]
```

**被推翻的旧假设**：`3角度 ×(RC/CR实虚)× 128通道`（角度在外、通道在内）。实测**正好颠倒**——通道在最外、4分量在最内。按旧假设做分组卷积会把数据彻底打乱。

**三重印证**：原始 matlab 源码 + 线一 reader 代码 + 物理自洽（baseline DAS 求和逻辑）。

**教训**：上一个 AI 的交接假设不可盲信。写架构前必须回溯最权威的数据源（原始 matlab）验证。

**路径**：`G:\rf_training_cache\dual_input_bmode\`

```
dual_input_bmode/
```
