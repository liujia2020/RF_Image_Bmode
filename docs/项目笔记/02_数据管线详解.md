## H5 源数据

与线一完全相同：1500 个 HDF5 文件（train 1050 / val 225 / test 225），每文件包含：

```
input:  F_RC_real/imag, F_CR_real/imag  [3角度, 128通道, 32Y, 32X, 64Z]
label:  DAS_target_avg_real/imag         [32Y, 32X, 64Z]   ← 33角度复数
baseline: DAS_input_avg_real/imag        [32Y, 32X, 64Z]   ← 3角度复数
meta:   z_idx/x_idx/y_idx                (空间索引)
```

**Python 读取**（`rf_learning_dataset.py`）：H5 `[角度,通道,Y,X,Z]` → transpose → `[Z,X,Y,通道,角度]` → reshape

> 📂 **原始 MATLAB 生成脚本已归档**至 [`docs/数据生成参考/`](../数据生成参考/)——H5 是怎么从 Verasonics 原始采集算出来的（波束形成、patch 提取、DAS 复合）全在那里。生成线二 cache 的是 `build_RF_random64_full1500.m`。

## 线二需要的三份派生数据

线二的双输入训练需要三份独立 cache：

### 1. RF input cache [1536, 64, 32, 32]

从 H5 的 input 组生成。流程与线一完全一致：
- `normalize=True`，scale = max(|label|)
- input / scale → fp16 memmap
- **独立 cache 路径，不读线一的 input.dat**

#### ⭐ 1536 通道内部顺序（权威，三重验证 — RF 支路第一层全看这个）

**1536 = 128 接收阵元 × 3 角度 × 4 分量**，C-order（最外/最慢 → 最内/最快）。通道 `c ∈ [0,1535]`：

```
comp = c % 4         # 0=RC_real 1=RC_imag 2=CR_real 3=CR_imag  （最快变）
ang  = (c // 4) % 3  # 0,1,2  三个发射角度
ch   = c // 12       # 0..127 接收阵元                          （最慢变）
等价：c = ch*12 + ang*4 + comp
[1536] 可无损 reshape → [128, 3, 4] = [阵元, 角度, 分量]
```

- **RC**=行发列收（接收孔径 ElementPos 129:256）、**CR**=列发行收（接收孔径 1:128），二者为正交平面波（OPW）互补采集。
- 3 角度 = matlab `angle_set` 顺序（值 `[3,38,73]` 是 alpha/beta 数组的**角度索引**、非度数本身；真实度数在 Verasonics TX，对架构无影响）。

**三重印证**：
1. **原始 matlab** `github.com/liujia2020/RF_Image` → `build_RF_random64_full1500.m`：`F_RC_patch/F_CR_patch : [Nz,Nx,Ny,128,N_angle]`，填充循环 `F_RC_patch(:,:,:,n_rx,ia)` 明示第 4 维=阵元、第 5 维=角度；H5 写入无 permute → h5py 读出维度反转 `[Nangle,Nch,Ny,Nx,Nz]`。
2. **线一 reader** `rf_learning_dataset.py`：`stack([RC_real,RC_imag,CR_real,CR_imag])`→transpose→reshape，与 matlab 链路吻合（线二 input.dat 由此 cp 而来）。
3. **物理自洽**：baseline DAS = F_RC、F_CR 各自 `sum over (通道, 角度)` 再平均（同脚本），坐实第 4 维=通道、第 5 维=角度。

> ⚠️ **推翻旧假设**：早期交接曾写 `3角度 ×(RC/CR实虚)× 128通道`（角度在外、通道在内）——**错**。实测通道在最外、4 分量在最内，布局完全相反；按旧假设做分组卷积会打乱数据。
> **双输入物理意义**：baseline B-mode = [128通道×3角度×2模态] 全部相干求和后的图；RF 输入保留这些**未坍缩**的细节 —— 这正是融合价值。

### 2. Baseline B-mode cache [1, 64, 32, 32] ← ⚠️ 线一没有，需新建

这是线二的**第二路输入**。从 H5 的 baseline 复数（DAS_input_avg_real/imag）转换。

**五步标准**（与 label B-mode 完全一致，REF 也相同）：
```
① 取包络:   abs(复数)             复数 → 幅值
② 归一化:   除以全局 REF 64407.58   幅值 → 0~1
③ log压缩:  20·log10(... + 1e-12)  → dB
④⑤ 截断:   限制在 [-60, 0] dB → 映射到 [0,1]
```

> ⚠️ 关键：baseline 和 label 用的是**同一个 REF**（64407.58），保证样本间一致。

### 3. GT B-mode cache [1, 64, 32, 32]

从 H5 的 label 复数（DAS_target_avg_real/imag）转换。五步同上，REF 同上。

线一已转过一份（路径不同），线二需要自己的独立副本。

## 训练时 DataLoader

线二需要自定义 Dataset reader，因为：
1. 读两个输入（RF + baseline B-mode），不是线一的一个
2. 返回 `{"input_rf", "input_bmode", "label_bmode", ...}`
3. `restore_scale=True`（RF 部分），B-mode 已是 [0,1] 不要乘 scale

## 全局 REF = 64407.58

从线一继承。来自 1500 个 label 包络的稳健高分位数，不被极端离群点绑架。
phantom 偏暗（均值 0.40）是全局参照的预期代价，本轮接受。

## 和线一的关键差异

| 项目 | 线一 | 线二 |
|------|------|------|
| Cache 路径 | Linux: `/home/liujia/rf_training_cache/...` | WSL: `G:\rf_training_cache\dual_input_bmode\` |
| Reader 输入 | 单输入（RF） | 双输入（RF + baseline B-mode） |
| baseline B-mode | 不进 cache，notebook 临时转 | 独立 cache，训练时直接读 |
| 训练代码 | `rf_cgan_models.py`（单编码器） | `rf_cgan_models.py`（双编码器） |
