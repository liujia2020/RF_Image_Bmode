# 06 — 关键概念速查

## 超声物理

### RF 信号
探头接收到的原始射频回波信号，在数 MHz 到数十 MHz 频段。本项目中的 RF 已经过
延迟对齐（delay-aligned），即不同通道的信号已按到达时间对齐。

### DAS (Delay-and-Sum)
传统的波束合成算法。核心思想：把 128 个通道的 RF 信号按各自的时间延迟加权求和，
重建出空间各点的反射强度。角度越多 = 求和次数越多 = 图像质量越好。

### 复值 IQ 数据
超声信号通常表示为复数值：`A·e^(iφ) = I + iQ`
- I (In-phase, 实部)：同相分量
- Q (Quadrature, 虚部)：正交分量
- 包络 (Envelope) = `√(I² + Q²)` = 信号幅度，决定了图像的亮暗
- 相位 = `atan2(Q, I)` = 信号的精细时序信息

本项目输入是复值 RF，输出是 B-mode 图像 [1, Z, X, Y]（包络+log压缩，值域 [0,1]）。
输入保留了复数/相位信息（RF 有正有负），但输出只取包络（放弃相位）。

### 散斑 (Speckle)
超声图像特有的颗粒状纹理，由组织中大量亚分辨率散射体的相干叠加产生。
- 不是噪声——散斑包含组织的微观结构信息
- Rayleigh 散斑：完全发育的散斑，SNR ≈ 1.91（幅度图）
- **散斑 SNR = mean / std**：衡量散斑"颗粒感"的指标
- 本项目 DAS label 散斑 SNR ≈ 2.0~2.2，属于正常范围

### 平面波成像
超声探头所有阵元同时发射，形成近似平面波。不同角度 = 不同发射方向。
- 3 角度：帧率高（快 11 倍），但图像质量差
- 33 角度：帧率低，但图像质量好
- 模型目标：3 角度入 → 33 角度出

## 深度学习

### 条件 GAN (cGAN)
普通 GAN：随机噪声 → 生成图像 → D 判断真假
条件 GAN：输入条件 +（可选噪声）→ 生成图像 → D 判断真假

本项目中：条件是 RF 信号 + baseline。类比 pix2pix，输入是"模糊图"，输出是"清晰图"。

### PatchGAN / Patch Discriminator
判别器不判整张图，而是输出一个 N×M 的分数矩阵，每个元素对应原图的一个 patch。
- 参数少（感受野小）
- 关注局部纹理（正好是散斑所在的尺度）
- 可以处理任意大小的输入图

本项目的 `Envelope2DPatchDiscriminator` 是 PatchGAN 的 2D 版本。

### LSGAN (Least Squares GAN)
用 MSE 代替 BCE（二分类交叉熵）作为判别器损失：
- BCE GAN：D 判"真(1)/假(0)"，sigmoid 交叉熵
- LSGAN：D 逼近 1（真）或 0（假），MSE 损失
- LSGAN 梯度更稳定，不易梯度消失

### L1 Loss 的问题
L1 = 逐像素取绝对值平均。在图像翻译中：
- L1 保结构好（大特征不容易变形）
- **但 L1 会模糊纹理**：因为最优解是条件均值，把多个可能的纹理模式"平均"掉了
- 本项目: 纯 L1 → "粥状散斑"就是这个原因

### 对抗损失补充 L1 的原理
- L1：保大结构，但模糊纹理 → `weight=10.0, 但用了低通, 不管高频散斑`
- 对抗：D 驱赶 G 产生逼真纹理 → `weight=1.0, 补上高频散斑的真实感`
- 载波：防止相位完全跑飞 → `weight=0.5, 微弱约束`

## 数据处理

### memmap (Memory-Mapped File)
NumPy 的内存映射机制。文件在硬盘上，但可以像 numpy 数组一样索引。
操作系统按需加载页到内存（page cache），不是一次性全部加载。

### HDF5 vs memmap

| | HDF5 | memmap |
|------|------|--------|
| 读速度 | 慢（分层索引+解压） | 快（连续裸二进制） |
| 灵活性 | 高（支持 group/dataset/attribute） | 低（纯数组） |
| 随机访问 | 支持但不高效 | 高效（offset 计算 O(1)） |
| 文件大小 | 较大（有元数据开销） | 最小（纯数据） |
| 适用场景 | 原始数据存储 | 训练用预处理缓存 |

### fp16 (半精度浮点)
- float32: 1 位符号 + 8 位指数 + 23 位尾数 = 4 bytes
- float16: 1 位符号 + 5 位指数 + 10 位尾数 = 2 bytes
- 数值范围: fp16 最大约 65504，min_normal ≈ 6×10⁻⁵
- 本项目：RF 数据范围在 ±6000，fp16 表示精度足够
- H2D 用 fp16（省一半 PCIe 带宽），计算用 fp32（autocast）

### Golden Values
一组固定条件下的参考数值，用于验证环境迁移后代码行为一致：
```
seed=20260605, indices=[0,350,700], y_idx=16
g_total=3.1978726387
```
如果在新环境跑出不同的数字，说明代码/数据/pytorch 版本有差异。

### Smoke Test
最小化的端到端测试：跑几步前向+反向，确认：
- 代码能跑（import 没问题）
- 不 OOM（batch size 适配正确）
- 不 NaN（数值稳定）
- D 和 G 的 loss 不是常数（对抗在运作）

不验证图像质量。

## 评估指标

### 散斑 SNR
```python
SNR = mean(envelope) / std(envelope)
```
- 完全发育的 Rayleigh 散斑 SNR ≈ 1.91
- 过于平滑的图像 SNR >> 2.2（散斑被"抹平"了）
- 过于嘈杂的图像 SNR << 1.9
- 本项目目标：pred_SNR 接近 label_SNR（~2.0）

### KS 统计量 (Kolmogorov-Smirnov)
衡量两个分布（pred 和 label 的包络直方图）之间的距离：
- KS=0 表示两个直方图完全一致
- KS=1 表示完全不重叠
- 本项目：env_hist_KS 用于比较 pred 和 label 的包络分布形状

### 人眼评审 (3D Slicer)
最终的、不可被代理指标替代的评估标准：
- 在相同的窗宽窗位下，candidate 和 DAS label 并列对比
- 检查散斑质地、结构保留、seam、伪影
- 三个状态：PASS / FAIL / INCONCLUSIVE
- 人眼 FAIL → run FAIL，无上诉权

## 项目特有术语

- **baseline**：用 3 个角度做传统 DAS 的低质量重建，用作模型输入条件和性能底线
- **carrier**：复值信号的载波分量（IQ 数据的复数本身，不是包络）
- **struct**：低通滤波后的包络，代表大尺度组织结构
- **seam**：patch 拼接时在边界出现的可见接缝，根因是归一化层统计不连续
- **Phase 2a**：损失设计的健康检查，不是图像质量验证
- **Codex**：执行侧 AI，负责写代码、跑实验、产出数据
- **Claude / Claude Web**：决策侧 AI，负责策略判断、风险分析、实验设计
