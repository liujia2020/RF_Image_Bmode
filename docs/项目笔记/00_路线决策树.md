# 00 — 路线决策树

> 线二路线图。相比线一的四层塔，线二起步路线简洁：只走双输入这条路。
> 线一的经验已告诉我们：纯 RF 容易 oversmooth、复数空间量纲地狱、对抗从未通电。
> 线二从这些教训出发，不重走弯路。

---

## 核心路线

```
输入: RF (3角度, 1536ch) + baseline B-mode (3角度 DAS 包络影像, 1ch)
       ↓
网络: 双支路编码器 + 融合 + 解码器 → B-mode [1, 64, 32, 32]
       ↓
损失: 0.84*(1-SSIM3D) + 0.16*L1（纯监督，不碰对抗）
       ↓
GT: 33角度 B-mode（REF=64407.58 五步标准转换）
       ↓
评估: Slicer 人眼终审 + SSIM/PSNR/CR/CNR/gCNR/FWHM
```

---

## 和线一的差异点

| 维度 | 线一 | 线二 |
|------|------|------|
| 输入 | 纯 RF | RF + baseline B-mode 图像 |
| 架构 | 单编码器 U-Net | 双支路编码器 + 融合 |
| 对抗 | 四轮空转后放弃 | 起点就不引入 |
| 训练空间 | 复数→后来切 B-mode | 起点就是 B-mode |
| 独特价值 | 纯 RF 端到端跳过 DAS | RF+图像融合的新范式 |

---

## 为什么这条路线

Hy-PCF 论文（IEEE TMI 2025）消融实验证明：
- 纯 channel data 模型 → oversmooth（正是线一的症状）
- 纯 image 模型 → 保留噪声/伪影
- **两者融合** → 即降噪又保纹理

线一已用 Tiny/Light3DUNet 验证了纯 RF 确实存在 oversmooth 问题。
线二直接走双输入，跳过了"证明纯 RF 不够"这个阶段（线一已证明）。
