## 核心架构：双输入编码器 + 融合 + 解码器

架构规格待 Claude 正式确定。初步方向（借鉴 Hy-PCF 思想，轻量化）：

```
输入1: RF [B, 1536, 64, 32, 32]
  → 1×1×1 Conv 降维 (1536→64) → 3D 编码器
                                     ↘
                                      融合（concat / attention）
                                     ↗
输入2: baseline B-mode [B, 1, 64, 32, 32]
  → 3×3×3 Conv 入口 → 3D 编码器
                                     ↓
                                  3D 解码器 → 输出 [B, 1, 64, 32, 32] + sigmoid
```

### 设计要点

- **融合点**：编码器各层均 concat？还是只在 bottleneck 融合？→ 待 Claude 决定
- **各向异性下采样**：Z 轴保守（多下一次），X/Y 可激进（min edge=32 → 最多 3 次 2× 下采样）
- **BN momentum=0.9**（线一已验证）
- **输出 bias 初始化 = -2.0**（防止 sigmoid(0)=0.5 灰雾起点）

### 线一 Light3DUNet 可作为起点

线一的 `Light3DUNet`(2.11M params) 已验证了各向异性下采样、BN、kaiming 初始化在 RF 数据上可行。线二可在此基础上加第二支路。

## 损失函数

与线一相同（与 Hy-PCF 论文相同）：

```python
loss = 0.84 * (1 - SSIM3D(pred, gt)) + 0.16 * L1(pred, gt)
```

- SSIM3D：单尺度，window=7，data_range=1.0，pytorch-msssim
- 全在 B-mode [0,1] 空间计算，量纲天然一致
- 无对抗、无判别器

---

## 关键约束（从线一继承）

1. **AMP 关**（真实幅值平方溢出 fp16）
2. **num_workers=0**（多进程+memmap 兼容问题）
3. **BatchNorm3d 不是 InstanceNorm3d**（防止拼图缝）
4. **normalize=True** + **restore_scale=True**（cache 对称）
5. **分层 batch 采样**（carotid/muscle/phantom 各 2，batch=6）
6. **全量验证禁止抽样**
7. **质量声称必经人类 Slicer 审查**
