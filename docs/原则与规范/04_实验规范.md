# 运行手册

最后更新: 2026-06-04 21:32:24 +08:00

## 1) 目的

本手册定义 RF-to-volume 实验如何创建、记录、审查和关闭。面向人类审查和 AI 辅助项目延续。

任何正式训练运行，必须先有运行文件夹和冻结配置，才能开始。

## 2) 运行文件夹标准

每次正式运行按三个阶段组织，一个阶段一个子文件夹：

```text
experiments/<track>/runs/<YYYY-MM-DD_short_name>/
│
├── 01_config/              训练前：配置与计划
│   ├── README.md            为什么做、假设、预期、最终状态
│   ├── config.yaml           冻结的训练配置
│   └── run_manifest.json     时间戳、git commit、环境信息
│
├── 02_train/               训练产物
│   ├── train.ipynb           ⭐ 核心：一个文件包含训练+验证全部信息
│   ├── checkpoints/          模型权重
│   └── probes/               每 N 步的 B-mode 探针切片
│
└── 03_validate/            验证产物
    ├── nii/                  NIfTI 产物
    ├── figures/              对比图
    ├── metrics/              指标 CSV
    └── verdict.md            人类审查结果与最终决定
```

三个阶段对应流程：先看配置 → 再看训练过程 → 最后看验证结果。

大型二进制产物（checkpoints、nii）可放 Git 外，但路径必须记录在 manifest 中。

## 3) 各文件说明

**01_config/README.md**：

- 为什么做这次运行。
- 单变量假设。
- 预期结果。
- 最终状态。

**01_config/config.yaml**：

- 训练的唯一冻结真实来源。
- 训练开始后若修改，应新建一次运行。

**01_config/run_manifest.json**：

- 时间戳、任务名
- git commit、dirty 状态
- 配置哈希
- 数据集路径、checkpoint 路径
- Python/CUDA/GPU 环境

**02_train/train.ipynb（强制 notebook 格式，禁止 .py 脚本）**：

一个文件贯穿到底。用户打开后从头跑到尾，所有信息都在，向下滚动即可浏览全部结果，不需要切换文件。

必须包含以下 Cell（按顺序），每个 Cell 必须有输出打印确认：

> ⚠️ 线二纯监督，无判别器。以下 cell 已按线二（单网络 + SSIM/L1）改写，去除线一 cGAN 残留。

Cell 1 — 启动自检
- 打印时间戳、实验名、模型真实类名 `type(model).__name__`
- assert 实验名与目录路径一致

Cell 2 — 配置打印
- 打印 config.yaml 完整内容
- 打印关键路径：cache路径、checkpoint路径

Cell 3 — 数据加载
- 创建 DataLoader（分层采样，每有效 batch carotid/muscle/phantom 各 2），打印每类样本数
- 取一个 batch，打印 input_rf / input_bmode / label_bmode 的 shape/dtype/max_abs/有无NaN

Cell 4 — 模型初始化
- 创建 DualBranchBModeUNet（线二纯监督，无判别器），打印参数量
- 确认 BatchNorm3d 数量（打印）、InstanceNorm3d=0、输出头 bias≈−2.0

Cell 5 — 训练循环（核心）
- 梯度累积：DataLoader 出 batch=6（分层每类 2），循环内拆 3×micro-batch=2，optimizer.step() 每 3 个 micro-batch 一次（等效 batch6，GPU 峰值 2 样本）。**AMP 关闭，全 fp32**。
- 每 epoch 打印: epoch | total_loss | ssim_term | l1_term | pred_mean | 耗时
- 每 epoch 输出探针三联图（保存到 probes/train/ 和 probes/val/）：
  固定 Y=16（中间层），XZ 切片，包络 B-mode
  窗宽窗位固定（与 label 共享 dB ref）
  Z 轴显示必须按物理 aspect ratio（根据体素间距计算：axial=0.0362mm, lateral=0.2mm, 显示时 aspect = (Nz×0.0362)/(Nx×0.2)），不可默认正方形
  **布局: baseline | pred | gt 横排**（列顺序与 Codex参考/执行规范.md 统一），标题标注 epoch=NNN category=XXX
- 每 epoch 输出散斑 SNR：pred_SNR 和 label_SNR，按 carotid/muscle/phantom 分列
- 每 epoch 追加一行到 stability.csv：epoch, total, ssim_term, l1_term, pred_mean, 三类 SNR
- 每 5 epoch 保存 checkpoint 到 checkpoints/checkpoint_epochNNN.pth

Cell 6 — 损失曲线
- matplotlib 子图：total_loss、ssim_term、l1_term（train + val 各一条）
- X轴 epoch，Y轴 loss，标题注明实验名

Cell 7 — 训练结束清理
- 只保留 best checkpoint（按 val total_loss 最低），删除其余
- 打印保留的 checkpoint 路径和对应的 epoch/loss

Cell 8 — 自动摘要
- markdown 格式，包含：
  实验名、训练时间、总 epoch 数
  best epoch 及其 total/ssim_term/l1_term
  最终 epoch 的三类散斑 SNR（pred vs label）
  pred_mean 演化（验证 bias=−2 防灰雾：训练初期应偏暗，非 0.5 灰雾）
  异常记录（有无 NaN/OOM/中断）

**02_train/probes/**：

- 训练中每 N 步导出的固定 slice/固定角度的 B-mode 探针图。
- 人类通过探针判断散斑形态是否朝 DAS 方向走，而非仅依赖损失数字。

**03_validate/verdict.md**：

- 人类视觉状态（PASS / FAIL / INCONCLUSIVE）。
- 指标状态。
- 最终决定。
- 什么失败了（如有）。
- 下一步动作。

## 4) 生命周期

1. 计划：写明运行目的和假设。
2. 冻结配置：写 `config.yaml`，记录 git 状态。
3. 训练：仅从冻结配置运行。
4. 导出材料：NIfTI、图表、指标。
5. 人类审查：用户检查 Slicer 材料。
6. 关闭：写 `verdict.md`，更新 `EXPERIMENTS_LOG.md`，然后提交。

## 5) 时间戳规则

每次运行和诊断输出以以下格式开头：

```text
YYYY-MM-DD HH:MM:SS +08:00 | TASK: <任务名>
```

相同的时间戳或生成的时间戳必须出现在 `run_manifest.json` 中。

## 6) 失败记录规则

失败是一等结果。记录：

- 预期什么。
- 实际发生了什么。
- 如何发现的。
- 是仅指标、仅视觉，还是两者。
- 结果是否推翻了一个假设。

不要将失败稀释为模糊的"需要改进"语言。

## 7) AI 角色边界

Codex 可以：

- 生成代码、配置、指标、图表、NIfTI 文件和摘要。
- 指出风险或不一致。
- 提出有证据的解释。

Codex 不可以：

- 在用户 Slicer 审查之前宣称图像质量通过。
- 将代理指标视为最终质量。
- 在没有运行文件夹和冻结配置的情况下开始新的正式训练运行。

## 8) 提交规则

一个有意义的动作，一次提交。提交信息必须描述实际变更。不要将训练代码、文档和大输出清理混在一次提交中，除非明确计划如此。
