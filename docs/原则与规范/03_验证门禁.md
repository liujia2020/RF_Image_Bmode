# 验证协议

最后更新: 2026-06-04 21:32:24 +08:00

## 1) 规则

质量声称必须经过验证门禁。仅靠代理指标永远不够。

如果人类在 NIfTI/Slicer 中的审查失败，则该次运行失败。指标不能推翻人类的失败判定。

Codex 负责产出材料和数字。用户负责签认视觉质量。

## 2) 正式运行必须产出的产物

每次正式运行必须提供：

- `label`、baseline 和候选模型的线性包络 NIfTI。
- 同一 volume 使用共享 label 参考的 dB NIfTI。
- 固定对比 PNG 图，方便快速审查。
- 标准指标 CSV。
- `verdict.md`，附人类审查状态。

NIfTI 的 spacing 必须为：

- z: `0.0362 mm`
- x: `0.2 mm`
- y: `0.2 mm`

NIfTI 数组顺序预期为 `[z,x,y]`，除非明确文档化。

## 3) 人类 Slicer 检查清单

对每个审查的 volume，在同一窗宽窗位下对比候选模型与 DAS label：

- 散斑形态：颗粒状 DAS 式纹理，不是模糊的糊状。
- 相干结构：血管壁、条带、界面、仿体结构保持完整。
- 点/线靶：label 中可见的目标不能消失。
- 幻觉：不能出现 label/RF 中不存在的新结构。
- 拼接缝：不能出现 patch 网格周期性线条。
- 深场：不塌缩、不失控过放大。

人类审查状态值：

- `PASS` — 通过
- `FAIL` — 失败
- `INCONCLUSIVE` — 待定

`INCONCLUSIVE` 必须在质量声称前有后续动作。

## 4) 标准指标

尽可能按类别报告：颈动脉、肌肉、仿体。

必需指标：

- 复数 L1 及相对 baseline 的改善。
- 包络 L1 及相对 baseline 的改善。
- 包络或 dB 包络的 PSNR，并注明精确定义。
- 均匀区域内的散斑 SNR，瑞利散斑 SNR 参考约 `~1.91`（适用时）。
- 包络分布的直方图距离或 KS 统计量。
- gCNR（有有效 ROI 时）。
- 拼接 volume 的缝指标。

点/线靶指标（适用时）：

- FWHM。
- 峰值保持。
- 旁瓣或对比度测量（若定义了 ROI）。

若 ROI 定义缺失，将指标标记为 `MISSING_ROI`，不能标记为通过/失败。

## 5) 验收策略

一次运行只有满足以下全部条件才能视为候选：

- 人类 Slicer 状态为 `PASS`。
- 保真度指标无严重退化。
- 无拼接缝退化。
- 散斑统计量合理且已文档化。
- 产物已归档在运行文件夹中，并汇总入 `EXPERIMENTS_LOG.md`。

以下情况必须拒绝或搁置：

- 人类审查发现 DAS 可见结构丢失。
- 散斑呈现糊状或幻觉。
- 点/线靶消失而 label 中存在。
- 结果仅改善代理指标。

## 6) 本协议背后的教训

WideDeep+SSIM 回归线在代理指标上看起来有希望，文档也曾过于乐观。后来人类 Slicer 审查将其否决。本协议的存在就是为了防止这种失败模式再次发生。
